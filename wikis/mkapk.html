Overview

mkapk is an efficient and lightweight Android build tool designed specifically for Termux. It provides a streamlined way to build Android applications directly on mobile devices without needing a desktop environment.

Primary Use Case: Mobile development on Android devices using Termux terminal emulator.

Key Features

  • ✅ Multi-language support: Java, Kotlin, C++, C
  • ✅ Lightweight and efficient
  • ✅ Optimized for Termux environment
  • ✅ Code obfuscation for release builds
  • ✅ Automatic optimization
  • ✅ Plugin system for additional languages
  • ✅ Fast compilation times
  • ✅ Easy configuration

Installation

Requirements

  • Termux terminal emulator
  • Android 7.0 or higher
  • At least 500MB free storage

Steps

  1. Install Termux from F-Droid or Google Play
  2. Clone the repository: git clone https://github.com/Nirbhay-Kumar-Developer/mkapk.git
  3. Navigate to directory: cd mkapk
  4. Run installer: ./install.sh

Quick Start

Basic Build

mkapk build

Release Build with Optimization

mkapk build --release --optimize

With Obfuscation

mkapk build --release --obfuscate

Supported Languages

Configuration

project.conf

[project]
name=MyApp
version=1.0.0
package=com.example.myapp

[build]
language=java
target_sdk=31
min_sdk=21

Frequently Asked Questions

Q: Can I use mkapk without Termux?

A: No, mkapk is specifically designed for Termux environment. However, you can use it on any Linux-based system with modifications.

Q: What is the difference between release and debug builds?

A: Release builds include obfuscation and optimization by default, while debug builds are faster to compile for development.

Q: How do I add custom plugins?

A: Place plugin files in the plugins/ directory and reference them in your configuration.
